## P0 — 建议下一迭代（2–4 周量级）

### 1. 存档 / 读档
- JSON 序列化：`month`、星图归属、势力资源、角色、联姻、事件 once 标记、RNG 种子  
- 主界面「存档槽 1–3」  
- **理由**：一局跨多回合，无存档难以传播口碑  

### 2. 教程剧本（强制 10 分钟）
- 锁定部分按钮，引导：移动 → 占无主 → 结束回合看 AI → 打一仗 → 处理事件  
- **理由**：网页访客跳失率高，需立刻懂「我该点什么」  

### 3. 事件池扩到 30+
- 只改 `data/events.json`：势力专属、战败后、联姻后、占密钥库后  
- 增加 `cooldown_months` 字段避免刷同一条  
- **理由**：表驱动，贡献者门槛最低  

### 4. 战棋最小内容包
- 2 种地形修正（星云减速、要塞防御）  
- 1 个远程单位（射程 2）  
- 战前「自动战报 / 手动战棋」每次可选  
- **理由**：差异化卖点，避免永远只有一种手感  

### 5. 移动端 / 小屏 UI
- 左右栏改为折叠；星图可 pinch/拖拽缩放  
- **理由**：外网链接大量来自手机

## P1 — 中期（像「小雷神」）

| 方向 | 内容要点 |
|------|----------|
| **身份完整** | 流民、掠航者开局 + 迁移规则（入狱/建国） |
| **科技树** | 4 分支 × 4 阶，合研条约 |
| **外交** | 同盟 / 破盟 / 从属 / 索贡 UI，而非只有联姻 |
| **政变** | 航官派系票数 + 倒计时 + 首都决战 |
| **势力 AI v2** | 效用函数：威胁评估、同盟倾向、不自杀式宣战 |
| **标准星图** | 30–40 节点，程序化 + 手摆关键点 |
| **模组加载** | `user://mods/*.json` 热更事件/角色 |

---

## P2 — 长期（产品化）

1. **多剧本**：弱势中兴 / 政变夜 / 黑旗建国 / 沙盒  
2. **程序化武将** + 20 手写关键角色线  
3. **音效 / 简短 BGM**（CC0 资源）  
4. **成就与图鉴**（占星、联姻、战棋连胜）  
5. **英文 i18n**（可选；当前产品定位简中）  
6. **深度战棋模式**（独立场景关卡编辑器）  
7. **异步多周目排行**（可选，需轻后端；开源可先不做）  

---

## 架构上建议坚持的原则

1. **战略层只吃 `BattleResult`** — 新战斗模式不渗进 `GameState` 细节  
2. **内容表驱动** — 事件、科技、舰种、联姻条款优先 JSON  
3. **Web 导出默认无线程** — 兼容 GitHub Pages（无 COOP/COEP）  
4. **简体中文玩家文案** — 代码标识英文  
5. **MIT 友好依赖** — 资源注明授权
